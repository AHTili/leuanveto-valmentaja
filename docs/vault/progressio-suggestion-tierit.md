# Suggestion-tierit — SAFE / TARGET / AGGRESSIVE (2D-δ)

**Status:** TOTEUTETTU-HEURISTIIKKA (v4.50.0 Track B 2D-δ; K-A2-monotonia deedc1a 2026-05-19; K3-4 historia-tietoinen VAROVAINEN retro-kenttä OBS-B2 2026-07-03)
**Lähteet:** Helms 2018 (spacing-suuruusluokka, RPE/RIR-autoregulaation ±-ikkuna); Akselin design-päätökset Q1–Q4 (2D-δ) + kenttähavainnot KH-5 (K-A2) ja OBS-B2 (K3-4) — tier-rakenteelle itselleen ei suoraa vertaisarvioitua lähdettä
**Koodiankkurit:** engine.js:4028–4050 (design-kommentti: spacing-perustelu + suppression + backward compat), engine.js:4051 (`generateSuggestions`), engine.js:4069–4071 (`SAFE_SPACING` = 0.015, `AGGRESSIVE_SPACING` = 0.015, `VX_OFFSET` = 1), engine.js:4092–4101 (K3-4 historia-cap), engine.js:4122–4128 (suppression-syyt), engine.js:4168–4184 (effectiveBias + default-valinta), engine.js:6289–6323 (recommend()-integraatio + `SUGGESTIONS_GENERATED`/`SUGGESTION_SUPPRESSED`-tracet), engine.js:6303 (`lastSessionDemonstratedKg: lastDemonstratedLoad`), engine.js:5247–5258 (jaettu demonstroitu-taso-skannaus), test-runner.js:5103 (`testAdaptiveSuggestions` T1–T14), test-runner.js:5192–5205 (T9 K-A2), test-runner.js:5248 (`testAdaptiveSuggestionsLearned` L1–L7)

Engine tuottaa 2–3 ehdotusta per sessio. Tier-arvot johdetaan aritmeettisesti nykyisestä TARGET-laskennasta — laskentaketju säilyy bittitarkasti, ja backward compat -sopimus (Akselin Q4 A) takaa että `rec.targetExternalLoad`/`targetVx`/`deltaPct` ovat aina TARGET-tierin arvot. Atletin valinta tallennetaan session-recordiin erikseen (`session.selectedSuggestionId`). Generaattorivirhe ei koskaan kaada recommend()-kutsua: catch-fallback palauttaa pelkän TARGETin (engine.js:6324–6342).

**Tier-aritmetiikka:** SAFE = TARGET-kuorma × 0,985 **samalla targetVx:llä**; AGGRESSIVE = TARGET-kuorma × 1,015, targetVx − 1 (lähempänä failurea). SAFE:n alkuperäinen design "+1 Vx" hylättiin **K-A2 e1RM-monotonia-invariantin** takia: +1 Vx pelkällä −1,5 %:n kuormalla *nostaa* Epley-e1RM:ää (KH-5-kenttähavainto: 69×5@V3 → e1RM 87,4 < "safe" 68×5@V4 → 88,4), jolloin varovainen vaihtoehto väittäisi korkeampaa kapasiteettia kuin tavoite. Ehto `safe.e1RM ≤ target.e1RM` vaatii saman Vx:n (VX_OFFSET = 0 SAFE:lle). T9-testin odotus jäi deedc1a:ssa päivittämättä ja korjattiin koodin suuntaan 2026-07-03 — esimerkki acceptance-session-not-summary-tyyppisestä testivelasta.

**K3-4 historia-tietoinen VAROVAINEN:** pelkkä −1,5 pp targetista ei erotu ("varovainen vain 1 kg kevyempi") ja voi progressioviikolla jopa *ylittää* viime session tason (kenttäcase: safe 165 kg = +5 kg vs. viime viikko). Siksi SAFE cappataan viime session demonstroituun kestotasoon: `lastSessionDemonstratedKg` on sama `lastDemonstratedLoad`-referenssi jonka SUSTAINABILITY_CAP käyttää (viimeinen target-Vx:n täyttänyt työsarja, fallback session-mediaani — skannaus on ehdoton, ei vain katon lauetessa). Cap toimii vain alaspäin: jos historia puuttuu tai on targetia korkeampi (paluuramppi), spacing-taso säilyy. `historyCapped`-lippu vaihtaa myös rationale-tekstin ("viime session näytetty taso, ei progressiota").

**AGGRESSIVE-suppression (Akselin Q1 B):** piilotetaan kun konteksti ei tue korkeampaa stimulusta — kuusi syytä: `readiness-cap` (capLevel ≥ 1), `recent-failure`, `grindy-bias` ([[readiness-grindy-bias-detektio]]), `rtf-not-reliable` (RTF-velocity-malli ei reliable-tilassa), `deload-phase` ([[progressio-deload-protokolla]]) ja `non-progression-day` (speed/competition). Jokainen syy kirjataan `suppressedReasons`-listaan ja `SUGGESTION_SUPPRESSED`-traceen audit-todistettavuuden vuoksi.

**Default-valinta (Q3 C + 2D-δ-C auto-learn):** capLevel ≥ 1 pakottaa SAFE:n ohittaen kaiken muun. Muuten `effectiveBias = biasBase + aggressivenessLearned`, jossa biasBase: stable −0,6 / balanced 0 / challenging +0,6, ja opittu säätö clampataan [−1; +1] (engine.js:4171–4173). Rajat: > +0,4 → AGGRESSIVE (jos saatavilla), < −0,4 → SAFE, muuten TARGET. Auto-learn siirtää biasta atletin valintahistorian mukaan ilman että eksplisiittinen `preferredBias`-asetus muuttuu — SAFE-streak painaa SAFE-suuntaan, TARGET-streak ei muuta mitään.

**8a-prior:** `aggressivenessLearned` on jo tuotannossa oleva opittava parametri: prior 0, kova koodiraja [−1; +1] (clamp engine.js:4172). Spacing-vakiot (1,5 pp) ovat design-vakioita, eivät tutkimusinvariantteja — jos niistä tehdään opittavia, ehdotetut rajat [0,01; 0,02] JA K-A2-monotonia on säilytettävä rakenteellisena ehtona (SAFE:n Vx-offset pysyy 0:ssa riippumatta spacing-posteriorista). Suppression-ehdot eivät ole opittavia: ne kytkeytyvät readiness- ja failure-invariantteihin.
